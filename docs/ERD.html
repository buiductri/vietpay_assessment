Integrity rules from the reasoning

Zero-sum

Each transaction's entries sum to zero (one CREDIT +amount, one DEBIT -amount, or more for FX). Enforced by a trigger or a check inside the DB transaction; a transaction that fails is rolled back.

Idempotency

The idempotency key is unique. A retried or duplicated request is rejected and mapped to the original transaction.

Kept to Section 2.1. FX is handled by multiple currency-pure wallets plus house exchange wallets (A1_VND, FX_VND, FX_USD, A2_USD), so it needs no extra entity. Holding wallets are ordinary wallets with a kind. Audit is handled at the application level and offloaded to MongoDB or another store, a separate section, so it is not a table here.
